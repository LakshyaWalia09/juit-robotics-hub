# JUIT Robotics Hub - Project Requirements & Dependencies

## Project Overview
JUIT Robotics Hub is a comprehensive web application for managing robotics club operations, including project tracking, faculty management, equipment showcases, and administrative dashboards with Supabase integration.

---

## Node.js & Package Manager Requirements

### Minimum Requirements
- **Node.js**: v18.0.0 or higher
- **npm**: v9.0.0 or higher (recommended) OR **Bun**: v1.0.0+ (used in this project)
- **Git**: v2.30.0 or higher

### Recommended Environment
- **Node.js**: v20.x LTS (latest stable)
- **Package Manager**: Bun (faster, more efficient)
- **OS**: Linux/macOS or Windows with WSL2

## Setup Instructions

### 1. Prerequisites Installation
```bash
# Install Node.js (v20.x recommended)
# Download from https://nodejs.org/

# Verify installation
node --version    # Should be v18+
npm --version     # Should be v9+

# (Optional) Install Bun for faster package management
# Visit https://bun.sh/
bun --version
```

### 2. Clone the Repository
```bash
git clone https://github.com/AdityaaSingh74/juit-robotics-hub.git
cd juit-robotics-hub
git checkout feature/website-improvements
```

### 3. Install Dependencies

**Using npm:**
```bash
npm install
```

**Using Bun (recommended for faster installation):**
```bash
bun install
```

### 4. Environment Setup
```bash
# Copy the example env file
cp .env .env.local

# Edit with your actual Supabase credentials
# The file already contains the project defaults, but update as needed
cat > .env << EOF
VITE_SUPABASE_PROJECT_ID=qcdnxeymnqlgwehonujj
VITE_SUPABASE_ANON_KEY=your_actual_key_here
VITE_SUPABASE_URL=https://qcdnxeymnqlgwehonujj.supabase.co
VITE_USE_MOCK=false
EOF
```

### 5. Run Development Server
```bash
# Using npm
npm run dev

# Using Bun
bun run dev
```

The application will be available at `http://localhost:5173`

### 6. Build for Production
```bash
# Using npm
npm run build

# Using Bun
bun run build

# Preview production build locally
npm run preview
```

### 7. Code Quality & Linting
```bash
# Run ESLint
npm run lint

# Fix linting issues automatically
npm run lint -- --fix
```

## Database Setup (Supabase)

### Connecting to Supabase
The project uses Supabase for backend services:

1. **Create Supabase Account**
   - Visit https://supabase.com
   - Create a new project
   - Get your Project ID and Anon Key

2. **Database Tables** (to be created)
   - `projects` - Robotics project tracking
   - `faculty_members` - Faculty information
   - `equipment` - Equipment inventory
   - `notifications` - Admin notifications
   - `admin_logs` - Audit trail for admin actions

3. **Authentication** (if implementing)
   - Supabase Auth is configured in `.env`
   - Implement via `@supabase/supabase-js`

4. **Real-time Subscriptions** (optional)
   - Enable Row Level Security (RLS) in Supabase
   - Configure real-time subscriptions as needed

## Troubleshooting

### Common Issues & Solutions

**Issue: Port 5173 already in use**
```bash
# Use a different port
npm run dev -- --port 3000
```

**Issue: Module not found errors**
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

**Issue: TypeScript errors**
```bash
# Rebuild TypeScript cache
npx tsc --noEmit
```

**Issue: Supabase connection fails**
- Verify `.env` credentials are correct
- Check internet connection
- Ensure Supabase project is active
- Try setting `VITE_USE_MOCK=true` to use mock data

**Issue: Tailwind styles not applying**
```bash
# Rebuild Tailwind CSS
npx tailwindcss -i ./src/index.css -o ./dist/output.css
```
